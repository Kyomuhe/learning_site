Object Clonning

Is a way of creating an exact copy of an object

In Java, object cloning refers to the process of creating a duplicate or copy of an existing object. This allows you to create a new object with the same state as the original object, but typically independent of it.

The clone() method is used to achieve object cloning in Java.

Here is how clonning works

  • Implement the Cloneable Interface: In order to enable cloning for a class, the class needs to implement the Cloneable interface. This interface acts as a marker interface, indicating that instances of the class can be cloned
  • Override the clone() Method: Once the class implements the Cloneable interface, it needs to override the clone() method from the Object class. This method is responsible for creating a copy of the object
  • Shallow Copy: By default, the clone() method creates a shallow copy of the object. This means that the fields of the object are copied, but if those fields themselves are reference types, only the references to the objects they point to are copied, not the objects themselves
  • Deep Copy: If you want to create a deep copy of the object, where not only the fields but also the referenced objects are copied, you need to implement custom logic within the clone() method to achieve this
