Object Interning

Its main aim is to store in the same memory location.

String interning is nothing but storing one copy of the distinct values, this is used to make string process more time and space efficient

There are two ways to create and assign values to strings as shown below

{{ code38 }}

Note:string constant pool is a collection of strings that are shared among multiple parts of the program when a string is created using string literals while heap memory is an area of memory where all objects including strings are stored, strings that are created using the new keyword or by modifying an existing string are always stored in the heap rather than in the string constant pool .

The method intern() creates an exact copy of the string object in the heap memory and stores it in the string constant pool

Note: if another string with the same contents exists in the string constant pool then anew object will not be created and the new reference will point to the other string provided the intern() method is used on both the strings
