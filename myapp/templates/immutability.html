Immutable Objects

Immutable objects are objects that cannot be changed, classes that are used to create immutable objects are called immutable classes,forexample string is an immutable class and once instantiated the value of the string object never changes.

They are final in nature

important Points to note about immutability:

  • Make the methods final and static
  • make the class final
  • All data fields must be final private
  • Initialise all fields using a constructor
  • immutable objects should not have getters and setters because these might cause privacy leakage

Privacy leakage is where a getter method leaks the privacy of a class to another class through inherritance

There are two types of immutability that is

  1. Strong Immutability : this is when the class is made final there for inherritance is imposible
  2. Weak Immutability : this is when the class is not final and so can be extended hence increasing chances of overriding and changing properties and methods
