Serialization

Serialisation in java is the process of converting the java code object into a byte stream to transfer the object code from one jvm to another and recreate it using the process of deserialization

Note: to serialize an object means to convert its state to a byte stream so that the byte stream can be reverted back into a copy of the object

To serialize an object, the parent class should satisfy the following

  • the class should implement the serializable interface
  • the class should have a no argument constructor

Remember that Serialization is the process of converting an object's state into a byte stream, which can then be saved to a file or transmitted over a network.

So lets the example above, The Student class is defined, and it implements the Serializable interface. This interface marks the class as serializable, allowing its objects to be converted to bytes for serialization.

Two constructors are defined: one with parameters to initialize the instance variables and another default constructor with no parameters

The objectSerialization method is defined within the Student class. This method takes a Student object as a parameter and performs the serialization process:

  • It creates a FileOutputStream to write bytes to a file named "student.ser".
  • It creates an ObjectOutputStream to handle the serialization of objects into the byte stream.
  • It uses the writeObject method of ObjectOutputStream to write the stu object to the byte stream.
  • The method then closes the output streams and prints a message indicating that the serialization is complete.

In the main method, an instance of the Student class named stu is created and initialized with student information

The objectSerialization method is called on the stu object to serialize it and save it to the "student.ser" file.
